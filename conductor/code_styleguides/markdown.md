# Markdown Style Guide

## 1. Headers
*   Use ATX style headers (`# H1`, `## H2`).
*   One blank line before and after headers.

## 2. Lists
*   Use hyphens `-` for unordered lists.
*   Use numbers `1.` for ordered lists.

## 3. Code Blocks
*   Always specify the language for syntax highlighting (e.g., ```python).
*   Use inline code `code` for commands, filenames, and variables.
